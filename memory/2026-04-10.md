### 本次修复的问题

#### 1. 设备台账列表空白（Equipment page empty）
- **现象**：`/equipment` 页面设备数据不显示
- **根因**：exec 工具每次调用后 `cd` 不保持！所有构建都在错误目录执行，导致 dist/ 内容没有更新
- **修复**：用绝对路径构建
  ```bash
  cd /workspace/smartsolar && rm -rf dist node_modules/.vite && npm run build
  ```
  之后 rsync 强制上传 `dist/` 全部内容到服务器
- **教训**：`cd` 在 exec 调用间不保持；用绝对路径而非相对路径

#### 2. 新建工单 stationId 默认值缺失
- **现象**：新建工单点击确定无反应（表单验证失败）
- **根因**：`handleAdd` 打开 Modal 时 `form.setFieldsValue({ stationId: undefined })`，导致 `stationId` 必填验证失败
- **修复**：`handleAdd` 时用第一个电站作为默认值
  ```ts
  form.setFieldsValue({ stationId: stations[0]?._id });
  ```

#### 3. 新建工单选择配件消耗无法选中
- **现象**：`mode="multiple"` 的备件 Select 无法选中
- **根因**：`form.getFieldValue('spareParts')` 在 `onChange` 内与 Select 形成循环：Select change → 触发 form.setFieldsValue → 触发 Select 重新渲染 → 再次触发 onChange
- **修复**：改用受控组件写法
  ```tsx
  // 本地状态替代 form.getFieldValue
  const [selectedSpareParts, setSelectedSpareParts] = useState<string[]>([]);
  // form 只在编辑时回填，不在手写 onChange 时更新
  <Select mode="multiple" value={selectedSpareParts} onChange={setSelectedSpareParts} .../>
  ```
- 同时 `handleSubmit` 需要正确将 ID 数组转为 `{sparePartId, quantity}` 格式提交
