## 晚间迭代续（20:00-20:52）

### 修复记录

#### 1. StationBuilder 保存失败
- **根因**：`status: 'planning'` 不在枚举 `['online','offline','maintenance']` 中；`location` 发送字符串但模型需要 `{ address, lat, lng }`
- **修复**：改为 `status: 'offline'`，`location` 改为 `{ address: stationData.location, lat: 0, lng: 0 }`
- 同时修复服务端路由 bug：`sparePartRoutes.ts` 中 `/:id` 在 `/consume-records` 之前，导致 `CastError` 500 崩溃（PM2 重启 16 次的原因）

#### 2. StationTopology 空白页
- **根因**：`station.location` 现在是对象 `{ address, lat, lng }`，直接渲染导致 React 崩溃：`Objects are not valid as a React child`
- **修复**：兼容 `location` 为字符串或 `{ address }` 两种格式

#### 3. 设备树 Equipment 页面内容为空
- **根因**：设备记录缺少 `categoryId` 字段（seed 数据没有正确赋值），导致树形结构每个分类下过滤结果为空
- **修复**：数据库迁移，为 equipment 集合中每条记录补全 `categoryId`
- **状态**：API 返回数据正确（categories 3条 + equipment 3条），Terry 那边可能浏览器缓存未刷新

#### 4. 告警转工单功能
- **已实现**，Alert 页面每行有「转工单」按钮，点击弹出 modal → POST `/api/work-orders`
- 不是缺失功能，只是 UI 上按钮不够显眼
